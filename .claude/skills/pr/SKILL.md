# PR Generator

이 프로젝트의 PR 초안을 작성하고, 사용자 확인 후 생성하는 스킬이다.

## 전체 흐름

```text
1. 현황 파악 → 2. 이슈 번호·라벨 자동 추출 → 3. 리뷰어 풀 구성
→ 4. PR 초안 작성 → 5. 사용자 확인 → 6. push + PR 생성
```

자동으로 추출 가능한 정보(이슈 번호, 라벨, 리뷰어)는 사용자에게 묻지 않고 처리하고,
confirm은 PR 초안을 보여준 뒤 딱 1번만 받는다.

외부에 영향을 주는 행위(`git push`, `gh pr create`)는 반드시 사용자가 승인한 이후에만 실행한다.

---

## STEP 1: 현황 파악 (병렬 실행)

```bash
git status
git branch --show-current
git log --oneline $(git merge-base HEAD origin/main)..HEAD
git diff --stat $(git merge-base HEAD origin/main)..HEAD
gh pr list --merged --limit 3   # 최근 PR 형식 감지
```

---

## STEP 2: 이슈 번호 · 라벨 자동 추출

브랜치 이름에서 이슈 번호를 추출한다.

```text
feat/#22-book-search  →  closes #22
fix/#105-login-error  →  close #105
```

이슈 번호를 추출했으면 연결 이슈에서 라벨을 가져온다:

```bash
gh issue view <이슈번호> --json labels
```

브랜치 이름에 이슈 번호가 없으면 `close #` 만 작성하고 넘어간다.

---

## STEP 3: 리뷰어 풀 구성

자기 자신은 리뷰어에서 제외한다:

```bash
ME=$(gh api user --jq .login)
```

리뷰어는 사용자가 별도로 지정하지 않는 한 생략한다.

---

## STEP 4: PR 초안 작성

아래 템플릿에 맞춰 초안을 작성한다.

```markdown
## 🧷 이슈

- close #<이슈번호>

## 🔨 작업 내용

<작업 내용>
```

### 작업 내용 작성 규칙

**형식**:
```text
- <큰 작업 단위>
  - <세부 작업 설명> (<커밋 해시 7자리>)
```

**예시**:
```text
- 인증 기능 구현
  - 사용자 세션 만료 시 자동 로그아웃 처리 (b3f91c2)
  - 로그인 실패 횟수 초과 시 계정 잠금 구현 (7d4e823)
- JWT 토큰 관리
  - JWT 재발급 엔드포인트 추가 (c19a051)
  - 소셜 로그인 콜백 URL 환경변수로 분리 (f82b374)
```

**규칙**:
- 계층은 2단계까지만 (큰 작업 단위 → 세부 커밋)
- 커밋 메시지의 prefix(`feat:`, `fix:`, `chore:` 등)는 제거하고 내용만 작성
- 커밋 해시는 7자리로 표기
- 관련 커밋은 큰 작업 단위로 묶어서 표현
- 부연 설명이 필요한 경우 3단계를 추가하는 대신 세부 작업 설명 안에 녹여서 표현
- merge 커밋 등 PR 내용과 무관한 커밋은 생략 가능
- 모든 설명은 간결하고 직관적으로 작성

초안을 출력할 때는 마크다운 코드블록 없이 바로 출력한다. 사용자가 복사해서 바로 붙여넣을 수 있어야 한다.

---

## STEP 5: 사용자 확인

초안 출력 후 반드시 아래와 같이 확인을 요청한다:

> 위 내용으로 PR을 생성할까요? 수정이 필요하면 말씀해 주세요.

사용자가 명시적으로 승인(예: "응", "ㅇㅇ", "올려줘", "맞아")하기 전까지 `git push` 및 `gh pr create`를 실행하지 않는다.

---

## STEP 6: push + PR 생성

사용자가 승인하면 순서대로 실행한다:

```bash
git push -u origin <현재 브랜치명>

gh pr create \
  --base main \
  --title "<브랜치명에서 유추한 제목>" \
  --body "<STEP 4에서 작성한 본문>"
```

---

## 안티패턴

- 사용자 확인 없이 `git push` 자동 호출 금지
- 사용자 확인 없이 `gh pr create` 자동 호출 금지
- 초안 없이 바로 PR 생성 금지
- 자동 추출 가능한 정보(이슈 번호, 라벨)를 굳이 사용자에게 질문 금지
